# Android 1.1 Release Prep

## Version

- `versionName`: `1.1`
- `versionCode`: `2`

Android version metadata now lives in `PiDayAndroid/gradle.properties` so release bumps do not require editing the app module script.

## Signing

1. Copy `PiDayAndroid/keystore.properties.example` to `PiDayAndroid/keystore.properties`
2. Fill in:
   - `storeFile`
   - `storePassword`
   - `keyAlias`
   - `keyPassword`
3. Run `cd PiDayAndroid && ./gradlew bundleRelease`

If `keystore.properties` is present, the release build will use that signing config automatically. If it is absent, Gradle keeps the release variant buildable without committed secrets. Use `assembleRelease` only for local APK smoke tests; the Play Store upload target should be the AAB from `bundleRelease`.

## Android 1.1 Feature Parity

- Nerdy Stats 2.0
- Date Battles
- Saved Dates leaderboard sorting, ranking, and label editing
- Classic and Nerd share styles plus battle-result sharing
- Free-search easter egg reactions

## Final Release Checklist

- Confirm `PIDAY_ANDROID_VERSION_NAME=1.1`
- Confirm `PIDAY_ANDROID_VERSION_CODE=2`
- Build `bundleRelease`
- Optionally build `assembleRelease` for device-side APK smoke testing
- Smoke test the release artifact on a device or emulator
- Upload the signed AAB to the Play Console
- Paste the `1.1` release notes into the Play Console "What's new" field
